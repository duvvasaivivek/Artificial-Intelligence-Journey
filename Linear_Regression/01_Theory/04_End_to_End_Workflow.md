# 04. End-to-End Workflow of Linear Regression

Machine Learning Workflow

1. Define the Problem
→ Clearly identify what needs to be predicted.

2. Collect Data
→ Gather historical examples.

3. Explore the Data (EDA)
→ Understand structure, missing values, outliers, and distributions.

4. Prepare the Data
→ Clean and preprocess the dataset.

5. Select Features & Target
→ Inputs = Features
→ Output = Target

6. Split the Dataset
→ Training Set (usually 80%)
→ Testing Set (usually 20%)

7. Train the Model
→ Learn the relationship between inputs and output.

8. Evaluate the Model
→ Check performance on unseen data.

9. Improve the Model
→ Feature engineering, more data, parameter tuning, or algorithm changes.

10. Deploy the Model
→ Use the trained model in real-world applications.

Key Terms

Dataset → Collection of examples
Training Set → Used to teach the model
Testing Set → Used to evaluate the model
Evaluation → Measure model performance
Deployment → Make the model available for users

Remember

Problem
→ Data
→ EDA
→ Preprocessing
→ Features & Target

→ Train
→ Evaluate
→ Improve
